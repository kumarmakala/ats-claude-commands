Implement a tenant-scoped, append-only audit log.

Audit events must include:

- event ID
- tenant ID
- actor type
- actor ID
- action
- resource type
- resource ID
- timestamp
- request/correlation ID
- result
- safe metadata
- previous-event hash or equivalent tamper-evidence mechanism when feasible

Audit:

- authentication events
- role changes
- department changes
- user lifecycle
- SSO and SCIM changes
- billing changes
- quota changes
- candidate uploads
- scoring
- pipeline transitions
- rejection
- exports
- retention changes
- erasure requests
- consent changes

Requirements:

- append only
- no resume content
- no unnecessary candidate PII
- tenant-scoped queries
- restricted audit.read permission
- retention policy
- export support
- detection of modification where technically feasible
