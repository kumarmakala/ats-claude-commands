Implement an automated candidate-assessment decision record.

For every scoring or recommendation capture:

- tenant
- job
- candidate/application
- scoring version
- model/version
- input document versions
- layer scores
- recommendation category
- generated explanations
- timestamp
- initiating user or job
- final human action
- human actor
- human reason when required

Rules:

1. ATS score cannot automatically reject a candidate.
2. Reject requires an authorized human confirmation.
3. UI and API must distinguish machine recommendation from human decision.
4. Reversal and correction must be recorded.
5. Historical decision records must not silently mutate.
6. Sensitive data must remain minimized.
7. Cross-tenant access must be impossible.

Add tests proving an automated worker cannot perform final rejection.
